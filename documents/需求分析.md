# 智能门锁项目需求分析文档

## 一、项目背景与目标

### 1.1 建设背景
- 传统机械锁存在钥匙易丢失、复制难管控、无法记录开锁行为等痛点。
- 现有电子锁功能冗余，用户对隐私及本地化控制要求更高，不希望过度依赖云端或复杂网络。

### 1.2 建设目标
- 提供**安全、便捷、可靠**的入户认证方式，支持人脸、指纹、密码、机械钥匙四种开锁方式。
- 实现本地用户权限分级管理（管理员/普通用户），具备低电量报警、防试错锁定、虚位密码等安全特性。
- 所有记录本地存储，不依赖云端，保障用户数据隐私。

### 1.3 适用范围
- 家庭住宅（入户门、室内门）
- 办公室、小型工作室
- 出租公寓（单户模式，无需集中管理）
- 酒店客房（本地管理，不联网）

## 二、用户角色分析

| 角色           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| **管理员**     | 最高权限。可添加/删除普通用户、修改管理员密码、重置门锁、查看本地记录。 |
| **普通用户**   | 仅具有开锁权限（人脸/指纹/密码），无法管理门锁设置或查看记录。 |
| **无临时用户** | 本系统不设临时用户、访客账户或时效性权限，所有用户均为长期有效。 |

> 注：运维人员（如电池更换、故障维修）由管理员或物理钥匙方式介入，无需独立系统角色。

## 三、功能性需求

### 3.1 开锁方式模块
- **人脸识别**  
  - 3D活体检测，防止照片/视频攻击。  
  - 识别速度 ≤ 0.5秒。  
  - 低功耗设计（影响待机时长）。  
- **指纹识别**  
  - 半导体或光学传感器，拒真率 ≤1%，认假率 ≤0.001%。  
  - 支持自学习算法。  
- **密码开锁**  
  - 固定密码（6-12位）。  
  - 虚位密码功能（前后任意输入数字，总长≤32位）。  
- **机械钥匙开锁**  
  - C级锁芯，防技术开启≥10分钟。  
  - 应急开锁备用，与电子系统物理隔离。

> 已移除：刷卡、NFC、手机远程开锁、蓝牙/Zigbee/Wi-Fi开锁。

### 3.2 权限管理模块
- **用户分级**：仅管理员与普通用户。  
- **管理员权限**  
  - 添加/删除普通用户的人脸、指纹、密码。  
  - 修改管理员密码、恢复出厂设置、导出本地记录（通过本地接口）。  
  - 设置系统时间（手动或自动校准）。  
- **普通用户权限**  
  - 仅可使用已录入的生物特征或密码开锁。  
  - 无权查看记录、修改门锁配置。  
- **权限管理方式**：须在门锁本地按键+管理员验证后操作，无远程下发或时段限制。

> 已移除：限定时段开锁、临时密码、临时用户、权限远程吊销。

### 3.3 安全与报警模块
- **低电量报警**  
  - 剩余电量 ≤10%时，门锁发出语音/蜂鸣提示，且每次开锁后提示。  
  - 若存在本地管理接口（如USB），可输出电量信息。  
- **多次试错锁定**  
  - 连续5次错误验证（任意方式组合），门锁锁定1~3分钟，期间拒绝所有开锁尝试，并记录事件。  
- **虚位密码**  
  - 在正确密码前后输入任意数字，确保他人旁观无法记录完整密码。

> 已移除：防撬报警、胁迫报警、防猫眼、防电磁干扰（保留物理结构防护）。

### 3.4 记录与查询模块
- **开锁记录**：包含时间、用户ID（管理员/普通用户）、开锁方式、成功/失败状态。  
- **报警记录**：仅限低电量、多次试错锁定两种事件。  
- **存储能力**  
  - 本地非易失存储，容量≥500条记录（循环覆盖）。  
  - **不支持云端存储**，**不支持远程查询**。  
- **记录查询方式**  
  - 管理员可通过门锁本地屏幕（如有）或连接管理终端（如USB线、串口）导出日志。  
  - 导出格式为CSV/TXT，需管理员密码授权。

> 已移除：撬锁报警事件记录、云端存储、远程App查询。

## 四、非功能性需求

### 4.1 性能需求
- 待机功耗 ≤ 50 μA（平均值），使用4节AA碱性电池应续航 ≥ 8个月（以10次/天开锁计）。  
- 开锁响应时间：从识别完成至电机转动 ≤ 1.5秒。  
- 指纹/人脸识别模块功耗：工作时 ≤ 200 mA，休眠时 ≤ 10 μA。  
- 本地记录存储至少1000条（可配置循环擦除）。

### 4.2 安全性需求
- **数据加密**：指纹特征、密码等生物及敏感信息在本地以加密形式存储（AES-128或国密SM4），不可直接读取。  
- **防重放/破解**：每次密码认证加入动态因子（如时间戳），防止抓包重放（但无网络时可简化为固定校验，由物理安全补偿）。  
- **锁芯等级**：C级锁芯，防钻、防撬、防拉。  
- **防静电**：接触放电 ±8 kV，空气放电 ±15 kV。  
- **紧急处理**：保留机械钥匙直开结构，断电或电子故障时可用钥匙开门。

### 4.3 可靠性与容错
- 断电后配置不丢失（Flash/EEPROM存储）。  
- 电机寿命：≥ 20万次开关动作。  
- 防尘防水等级：室内使用 IP52，若用于半户外（有遮雨）需 IP54。  
- 工作温度：-25℃ ～ +70℃；工作湿度：20% ～ 93% RH。  
- 电源管理：支持电池反接保护，低电压时确保至少可开锁50次。

### 4.4 可扩展性
- **不强制要求无线模块**，但可预留硬件接口（如USB）用于固件升级或日志导出。  
- 若日后需加入智能家居联动，可更换主控板版本（本次不实现）。  
- 支持不同锁体类型（标准6068锁体、霸王锁体等）的机械适配。

### 4.5 易用性
- 语音/蜂鸣提示：开锁成功/失败、低电量、操作指引。  
- LED指示灯：不同颜色表示不同状态（蓝色待机、绿色成功、红色失败）。  
- 电池仓设计：正负极标记清晰，防止装反。  
- 室内紧急快开：旋转把手或下压一次即可开门，无需电机驱动。  
- 管理员操作界面简洁（按键+菜单）。

## 五、接口需求

> 由于去除了蓝牙/Wi-Fi/Zigbee等无线开锁功能，接口仅限本地物理及调试用途。

| 接口类型         | 用途                                                         |
| ---------------- | ------------------------------------------------------------ |
| **USB Type-C**   | 可选，用于本地日志导出、固件升级、电量读取（需管理员授权）。 |
| **串口 (UART)**  | 生产调试、故障诊断，外部不开放。                             |
| **电机控制接口** | 内部驱动直流/步进电机，定义电压、时序逻辑。                  |
| **传感器接口**   | 门磁信号（选配）、锁舌状态检测。                             |

> 无任何与手机App、云端平台、第三方远程系统的接口。

## 六、环境与部署需求

### 6.1 硬件环境
- 主控芯片：低功耗ARM Cortex-M系列（如STM32L0）。  
- 人脸识别模块：3D结构光或双目红外，带独立NPU。  
- 指纹模块：电容式传感器。  
- 电机：小型减速电机，扭矩 ≥ 2.5 N·m。  
- 电池：4节AA碱性电池（或锂铁电池）。  
- 锁体材质：不锈钢或锌合金。

### 6.2 工作环境
- 温度：-25℃ ～ +70℃（宽温设计）。  
- 湿度：20% ～ 93% RH（无凝露）。  
- 抗震动：满足家用及办公环境。

### 6.3 安装要求
- 适用门厚：35 mm ～ 55 mm（标准门），可加配件至70 mm。  
- 导向片尺寸：标配240×24/30 mm 等常用规格。  
- 安装方式：替换原有机械锁，无需重新开孔。

### 6.4 联网方式
- **无联网能力**（独立门锁）。所有配置、记录均本地完成。

## 七、数据需求

### 7.1 数据实体
- **用户信息表**：字段包括 UserID（1字节）、姓名（可选，本地仅存储ID）、人脸特征模板、指纹特征模板、密码（Hash+salt）、用户类型（0=管理员，1=普通用户）。  
- **开锁记录表**：字段包括 时间戳（UTC+local）、UserID、开锁方式（1=人脸/2=指纹/3=密码/4=机械钥匙）、结果（成功/失败）。  
- **报警事件表**：字段包括 时间戳、事件类型（1=低电量，2=多次试错锁定）、详情。  
- **设备信息表**：固件版本、序列号、电池电量百分比、总开锁次数等。

### 7.2 数据存储与备份
- 所有数据存储于本地Flash，容量≥4 MB（满足1000条记录+特征模板）。  
- 无云端备份。管理员可通过USB导出CSV文件进行本地备份。  
- 恢复出厂设置时清空所有用户数据及记录，但保留出厂日志。

### 7.3 隐私保护
- 人脸、指纹等生物特征**绝不上传任何外部设备**，仅本地存储且加密。  
- 密码不以明文存储，使用PBKDF2或类似算法加盐Hash。  
- 通过USB导出记录时不包含生物特征，仅含脱敏后的UserID。

## 八、约束与假设

### 8.1 法规合规
- 符合《电子防盗锁》GA 374-2019标准。  
- 符合《个人信息保护法》要求（生物信息本地处理）。  
- 取得CCC认证（中国强制性产品认证）及SRRC（若含无线模块，但本设计无无线，故不适用）。  
- 出口版本需符合当地EMC及安全标准。

### 8.2 成本与电池寿命平衡
- 高性能人脸识别模块功耗较高，需选用低功耗3D方案，并在不使用时完全断电。  
- 预期BOM成本控制在家用市场可接受范围（约300-600元人民币）。

### 8.3 机械兼容性
- 需适配市面上80%的防盗门锁体（标准6068），并提供多种导向片。

### 8.4 不包含的功能（明确排除）
- 远程开锁、临时密码、时效权限、联网管理、云端备份、手机消息推送。

## 九、附录

### 9.1 术语表
- **虚位密码**：在正确密码前后任意输入若干数字，用于防窥视。  
- **C级锁芯**：国家锁芯安全等级中的最高级，防技术开启时间≥10分钟。  
- **拒真率（FRR）**：合法用户被拒绝的概率。  
- **认假率（FAR）**：非法用户被接受的概率。  
- **3D结构光**：通过红外点阵投射获取深度信息的人脸识别技术，抗平面攻击。

### 9.2 参考标准
